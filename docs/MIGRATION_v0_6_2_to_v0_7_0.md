# RiskRadar v0.6.2 → v0.7.0 마이그레이션

## 성격

v0.7.0은 **변화 중심 UI 릴리스**입니다. v0.6.2의 `decision_snapshot.json`·`decision_diff.json`을 그대로 사용하며 판정 규칙, FRED 수집, 캐시 schema, Telegram, GitHub Actions workflow를 바꾸지 않습니다.

## 핵심 변경

1. 상위 탭 8개를 5개로 통합
   - 한눈에 보기
   - 기업 신용
   - 흐름과 차트
   - 비교
   - 지표 설명
2. 첫 화면에 오늘 한 줄, 최근 갱신 변화, 아직 남아 있는 변화, 다음 확인을 우선 표시
3. 핵심 6개를 모바일 카드로 표시하고 `변화 있는 항목만 보기`를 기본값으로 사용
4. 기업 신용 범위를 HY·BBB·A·CP 지도 형태로 표시
5. 데이터 상태는 상단 요약 + 전역 아코디언으로 이동

## 데이터 호환

- v0.7.0 UI는 v0.6.2가 생성한 데이터와 호환됩니다.
- v0.6.2보다 오래된 데이터에는 권위 있는 diff가 없으므로 `최근 갱신에서 달라진 것`이 비어 있을 수 있습니다. 최신 refresh 후 정상 배치가 두 번 쌓이면 비교가 시작됩니다.
- 백필은 하지 않습니다.

## 배포

전체 패키지로 교체하거나 patch-only 파일만 덮어쓸 수 있습니다. workflow 파일은 변경하지 마십시오. v0.7.0은 UI 변경이므로 배포 직후 새 refresh가 필수는 아니지만, 최신 v0.6.2 이상 권위 기록을 읽는 것이 권장됩니다.
