# RiskRadar v0.4.4 → v0.4.5 마이그레이션

## 한눈 요약

v0.4.5는 새 분석축을 추가하는 버전이 아니다. **기존 v0.4.4의 분석 구조를 유지하면서, 사용자가 실제 화면을 읽기 어렵게 만들던 전문용어와 데이터 누락 대응을 정리한 버전**이다.

핵심 변경은 세 가지다.

1. 화면 용어를 쉬운 말로 전면 교체
2. `최근 5년/10년 위치`를 확률처럼 보이지 않게 순위 감각으로 표시
3. 보조 확인 지표 일시 수집 실패 시 재시도 + 직전 성공값 유지

경제일정·이벤트 기능은 추가하지 않았다. GitHub workflow 파일도 수정하지 않았다.

## 2. `최근 5년/10년 위치` 표시 변경

이전:

```text
최근 5년 위치 82%
```

문제:

- 위험확률 82%처럼 오해하기 쉽다.
- 퍼센트가 무엇을 뜻하는지 화면만 보고 알기 어렵다.

v0.4.5:

```text
최근 5년과 비교
상위 18% 수준 (82%의 날보다 높음)
```

뜻:

> 현재 값이 최근 5년 관측일의 82%보다 높았다는 뜻이다. 위험확률이나 하락확률이 아니다.

낮은 쪽은 예를 들어:

```text
하위 20% 수준 (20%의 날보다 높음)
```

으로 표시한다.

## 3. 쉬운 용어 사전 추가

`현재 상황` 탭에 `용어가 어렵다면 먼저 보기` 아코디언을 추가했다.

아래 개념을 한 줄 설명과 함께 제공한다.

- 저신용 회사채 추가금리
- 우량 회사채 추가금리
- 채권시장이 보는 10년 물가전망
- 장기채 보유 추가보상
- 10년 실질금리

`추가금리`가 기업의 실제 대출금리 자체가 아니라 **미국 국채보다 얼마나 더 높은 금리를 요구받는지**라는 점도 명시한다.

---

## 4. 동적 해석문 전문용어 순화

정적 지표명만 바꾼 것이 아니다. 기존 캐시와 조합 해석에 남아 있던 표현도 화면 출력 직전에 쉬운 말로 바꾼다.

예:

- 명목금리 → 일반 국채금리
- 인플레이션 보상 → 물가 관련 금리 요인
- 정책경로 → 기준금리 예상 경로
- 경기 사이클 → 경기 흐름
- 광범위한 → 넓은 범위의
- 디스인플레이션 → 물가상승률 둔화
- 위험보상 → 추가 보상
- 보합 → 뚜렷한 변화 없음

내부 계산 문구는 유지할 수 있으나 최종 사용자 화면에는 쉬운 표현을 우선한다.

## 5. `우량 회사채 추가금리` 데이터 없음 보완

FRED series id는 기존과 동일하게 `BAMLC0A0CM`을 사용한다.

v0.4.4까지는 보조지표 한 번의 수집 실패가 해당 배치의 `확인 불가`로 바로 이어질 수 있었다.

v0.4.5에서는:

1. FRED 일시 오류·429·5xx·네트워크 오류를 짧게 재시도
2. 그래도 실패하면 직전 성공한 보조지표 값 검색
3. 직전 성공값이 있으면 최신성을 다시 계산해 이어 사용
4. 화면에 `이번 수집 실패로 직전 성공값 사용` 표시
5. 너무 오래된 값은 현재 해석 지지·약화 집계에서 자동 제외

### 중요한 한계

현재 활성 캐시에 해당 지표가 없고 과거에도 한 번도 성공 저장된 값이 없다면, 코드 배포만으로 새 데이터가 생기지는 않는다. **v0.4.5 배포 후 한 번의 성공 refresh가 필요하다.**
