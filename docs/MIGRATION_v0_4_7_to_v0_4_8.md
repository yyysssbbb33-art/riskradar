# RiskRadar v0.4.7 → v0.4.8

## 핵심 변경

### 1. `마지막 데이터 생성 코드 버전: 기록 없음` 의미 명확화

이 값이 비어 있다고 데이터 자체가 없는 것은 아니다. 활성 캐시의 `status.json`과 `data_quality.json`에 `code_version` 메타데이터가 없는 구버전 데이터라는 뜻이다.

v0.4.8 데이터 상태 탭은 아래를 함께 보여준다.

- 현재 화면 코드 버전
- 활성 데이터 버전
- 마지막 데이터 생성 시각
- 데이터 생성 코드 버전
- 실제로 읽고 있는 Dataset 저장소

현재 화면은 최신인데 데이터 생성 코드 버전이 없으면, 최신 GitHub 배치가 아직 현재 Dataset에 새 캐시를 발행하지 않았거나 옛 배치 데이터를 읽고 있을 가능성을 우선 확인한다.

### 2. 지난 30일 흐름을 저장 스냅샷 없이 즉시 재구성

기존 구현은 `versions/<cache_version>/signal_matrix.parquet`가 매일 쌓여야 지난 30일 흐름이 풍부해졌다. v0.4.8부터는 최신 캐시의 `chart_data.parquet`에 이미 들어 있는 핵심 6개 지표 과거 시계열을 사용한다.

```text
최신 chart_data
→ 최근 30일 실제 관측일 필터
→ 값·1개월/3개월 변화·과거 위치·상태 복원
→ 지난 30일 흐름 해석
```

따라서 앱을 오늘 처음 배포했거나 저장 스냅샷이 1개뿐이어도 한 달 흐름이 바로 표시된다.

주의:

- 날짜는 RiskRadar 저장일이 아니라 실제 지표 관측일이다.
- 당시 화면을 보존한 기록이 아니라 현재 코드 규칙으로 과거 관측일을 다시 읽은 결과다.
- 보조지표 3개는 현재 구조상 최신 방향 요약만 저장하므로, 이번 버전에서 완전한 30일 과거 경로 재구성 대상은 핵심 6개다.
- `chart_data`가 없는 구버전 캐시에서만 기존 저장 스냅샷을 fallback으로 사용한다.

## 테스트

- 과거 chart_data에서 30일 기록 재구성
- 재구성 표가 저장일이 아닌 관측일을 표시
- code_version 누락을 구버전 메타데이터로 판별
- 기록된 배치 코드 버전 우선 표시
