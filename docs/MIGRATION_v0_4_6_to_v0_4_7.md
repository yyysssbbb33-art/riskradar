# RiskRadar v0.4.6 → v0.4.7 마이그레이션

v0.4.7은 사용자 요청 4가지를 한 번에 반영한다.

1. 과거 위치 표현을 `상위 N% 구간 / 하위 N% 구간`으로 단순화
2. `BAMLC0A0CM`(신용등급 높은 기업의 추가금리) 수집·복구·진단 강화
3. FOMC·고용·물가·장기국채 관련 정적 결과별 해석 가이드 추가
4. 기존 `최근 30일`을 `지난 30일 흐름`으로 확장

경제일정 API·스케줄 수집·뉴스 자동판독은 추가하지 않았다. `refresh.yml`도 수정하지 않았다.

---

## 1. 과거 위치 표현

기존:

```text
낮은 편 · 13%의 날보다 높음
높은 편 · 82%의 날보다 높음
```

변경:

```text
하위 13% 구간
상위 18% 구간
중간 구간
```

뜻:

- `상위 18% 구간`: 과거 관측값을 낮은 값부터 높은 값까지 줄 세웠을 때 높은 쪽 18% 안
- `하위 13% 구간`: 낮은 쪽 13% 안
- 위험확률·상승확률·하락확률이 아님

예전 캐시에 `최근 10년 위치 82%`처럼 저장된 문구도 화면 출력 시 `최근 10년 중 상위 18% 구간`으로 바뀐다.

## 2. 신용등급 높은 기업의 추가금리 수집 강화

실제 수집 시리즈는 기존과 동일하다.

```text
FRED series: BAMLC0A0CM
internal key: IGOAS
```

### 확인 결과

수집 로직은 원래 존재했다. 다만 두 문제가 있었다.

1. 실제 코드에는 지표별 재시도가 없었음
2. 실패 시 `last_good_aux()`가 최신 활성 캐시 하나만 확인했음

따라서 최신 캐시에 IGOAS가 없으면 더 오래된 버전에 정상값이 있어도 찾지 못했다.

### v0.4.7 변경

- 보조지표마다 최대 2회 FRED 요청
- 현재 수집 실패 시 과거 Dataset 버전을 최신→과거 순서로 검색
- `fetch_status=ok`이며 실제 값이 있는 마지막 **진짜 성공 행**만 복구
- `carried_forward` 행을 다시 성공값으로 취급하지 않아 실패값 연쇄 복사를 방지
- 정상 수집이면 과거 버전 검색 자체를 하지 않음
- 복구값이 오래되면 기존처럼 해석에서 제외

### 데이터 상태 탭

보조지표별로 아래를 표시한다.

- 쉬운 지표명
- FRED 시리즈 ID
- 정상 수집 / 과거 마지막 성공값 사용 / 실패
- 사용 중인 관측일
- 최신성
- 오류 내용

또한:

```text
현재 화면 코드 버전
마지막 데이터 생성 코드 버전
```

을 함께 표시해 HF Space 코드와 실제 배치 코드가 다른 경우를 확인할 수 있다.

## 3. 앱 밖에서 확인하면 좋은 발표·뉴스

각 핵심 지표의 `상세 설명 보기` 안에 새 섹션을 추가했다.

```text
앱 밖에서 확인하면 좋은 발표·뉴스
```

일정은 수집하지 않는다. 아래 흐름만 제공한다.

```text
왜 보나
→ 무엇을 볼까
→ 어떤 결과면 어떤 설명이 강해지나
→ 결과가 엇갈리면 무엇을 말할 수 없나
```

### 포함한 발표·뉴스

- FOMC(Fed 금리회의)
- 미국 고용보고서
- 미국 물가 발표(CPI·PCE)
- 미국 장기국채 발행·입찰 소식

### 예: FOMC

- 앞으로의 금리 예상 하락 + 실업률 전망 상승
  - 경기 둔화 대응형 금리인하 기대 설명 강화
- 앞으로의 금리 예상 하락 + 물가 전망 하락 + 고용 전망 안정
  - 경기 급락보다 물가 상승세 둔화 속 금리 부담 완화 설명 강화
- 앞으로의 금리 예상 상승 + 물가 전망 상승
  - 높은 금리가 오래 이어질 가능성 설명 강화
- Fed 내용과 2년 금리 반응이 반대
  - 이미 가격에 반영됐는지, 기자회견의 다른 내용이 더 중요했는지 확인

### 예: 고용보고서

- 일자리 증가 둔화 + 실업률 상승 + 이전치 하향 수정
  - 경기 둔화형 금리인하 기대 설명 강화
- 일자리 강함 + 실업률 안정 + 임금 강함
  - 빠른 금리인하 설명 약화
- 일자리 강함 + 실업률 상승
  - 결과 충돌. 노동시장 참여·이전치 수정 추가 확인

## 4. `최근 30일` → `지난 30일 흐름`

새 탭을 추가하지 않고 기존 탭을 확장했다.

상단에 자동 해석을 추가한다.

### 한 달 사이 어떻게 달라졌나

핵심 지표의 30일 전→현재 변화를 요약한다.

### 한때 크게 움직였다가 되돌아온 것

시작점과 현재는 비슷하지만 기간 중 큰 움직임이 있었던 경우를 잡는다.

예:

```text
VIX 18 → 기간 중 35 → 현재 18
```

단순 시작·끝 비교로 `변화 없음` 처리하지 않는다.

### 아직 남아 있는 변화

기간 시작보다 현재 값이 달라진 상태가 남아 있는 지표를 보여준다.

### 움직인 순서

기간 중 기본상태→변화상태로 처음 전환된 날짜를 비교한다.

순서는 원인관계가 아니라 `무엇이 먼저 눈에 띄었는지`만 보여준다.

### 금리는 어떻게 움직였나

2년·30년 금리의 한 달 방향을 함께 본다.

- 2년↓ · 30년↑
- 2년↑ · 30년↓
- 둘 다 상승
- 둘 다 하락

### 지금부터 확인할 것

30일 과정과 현재 보조지표를 연결한다.

예:

- VIX는 되돌아왔지만 낮은 등급 기업 추가금리는 높은 상태 유지
  - 높은 등급 기업 추가금리도 오르면 부담 확산 설명 강화
  - 그대로/하락이면 낮은 등급 기업 쪽 집중 설명 강화

- 2년↓ · 30년↑
  - 10년 물가 예상 상승 → 물가 요인 설명 강화
  - 장기채 추가 보상 상승 → 장기채 수요·공급/보유 부담 설명 강화
