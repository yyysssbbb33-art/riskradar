# RiskRadar v0.5.1 → v0.6.0 마이그레이션

## 1. 이번 업그레이드의 성격

v0.6.0은 단순 지표 추가가 아닙니다. 기업 신용 영역을 `현재 값과 방향` 중심에서 **범위·지속·잔존 변화** 중심으로 확장하고, v0.5.x에서 한 묶음이었던 지표 역할을 3층으로 다시 정리합니다.

```text
핵심 신호
→ 기존 핵심 6개

기업 신용 렌즈
→ HY-BBB

확인 지표
→ 물가 예상 / 장기채 추가 보상 / BBB / A / CP

외부 참고
→ NFCI / STLFSI
```

기업 신용 범위 엔진의 실제 시장 노드는 `HY / BBB / A / CP`입니다. HY-BBB는 독립 노드가 아니라 등급 차별을 읽는 렌즈입니다.

## 2. 가장 안전한 업로드 방법

### 권장: patch-only ZIP 사용

`riskradar_v0.6.0_patch_only.zip`의 파일만 기존 GitHub 저장소에 덮어쓰는 방법을 권장합니다.

patch-only에는 workflow 파일이 없습니다.

따라서 실제 운영 중인 다음 파일을 건드리지 않습니다.

- `.github/workflows/refresh.yml`
- `.github/workflows/sync-to-hf-space.yml`

### full ZIP 사용 시

full ZIP에는 현재 기준 workflow 파일도 포함돼 있지만, 실제 운영 저장소의 workflow를 따로 수정해둔 경우에는 기존 파일을 유지하는 편이 안전합니다.

이번 v0.6.0 개발에서는 workflow 자체를 수정하지 않았습니다.

## 3. 환경변수·Secret 변경

**새 Secret이나 환경변수는 필요하지 않습니다.**

기존 설정을 그대로 사용합니다.

- `FRED_API_KEY`
- `CACHE_BACKEND`
- `HF_TOKEN`
- `HF_DATASET_REPO_ID`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

## 4. 데이터 수집 변경

### 새로 일반 사용자 기능에 사용하는 지표

- `BAMLC0A3CA` — A등급 기업의 추가금리

### 계속 수집하지만 일반 상세 목록에서 숨기는 지표

- `BAMLC0A0CM` — broad IG 전체 평균

이 지표는 v0.5.x 캐시 호환과 운영 진단을 위해 수집 경로는 유지하지만, BBB를 포함하므로 기업 신용 범위 엔진의 독립 노드로 사용하지 않습니다.

### 실제 신용 시장 노드

- HY — 기존 핵심 `BAMLH0A0HYM2`
- BBB — `BAMLC0A4CBBB`
- A — `BAMLC0A3CA`
- CP — `RIFSPPNA2P2D30NB - RIFSPPNAAD30NB`

## 5. ICE 회사채 자료 범위 변경

v0.6.0은 ICE BofA 계열의 현재 FRED 제공 범위를 장기 역사처럼 과장하지 않습니다.

### HY

기존:

- 최근 5년 위치
- 최근 10년 위치

v0.6.0:

- 최근 3년 위치
- 실제 5년·10년 달력 자료가 없으면 해당 위치는 계산하지 않음

관측치 개수만 많다고 3년 자료를 `5년 위치`로 표시하던 가능성을 막기 위해 실제 달력 커버리지를 검사합니다.

### BBB·A

상세 화면에서는 `현재 확보된 자료 범위 안 위치`라고 표시합니다.

다음 표현은 사용하지 않습니다.

- 장기 역사상 이례적
- 과거 위기 수준
- 최근 10년 기준

## 6. 새 기업 신용 범위·지속 엔진

### 1차 산출물

- 현재 에피소드 존재 여부
- 실제 참여 시장
- 각 노드의 현재 상태
- 기준선 대비 아직 남은 변화
- HY-BBB 등급 차별 렌즈
- VIX 시작 무렵 맥락

### 노드 상태

- 평소 상태
- 초기 변화
- 새로 상승
- 상승 지속
- 되돌림
- 정상화

`초기 변화`는 공식 참여 범위에 포함하지 않습니다.

### 에피소드 상태

- 활성
- 휴면
- 종료

휴면은 변화가 완전히 사라졌다는 뜻이 아닙니다. 이전 변화의 잔존은 남아 있지만 최근 새 참여나 의미 있는 확대·상태 전환이 없다는 뜻입니다.

### 기준선 동결

변화가 확정되는 순간 다음 정보를 고정합니다.

- 추정 시작일
- 확정일
- 시작 직전 기준선

이후에는 소급 수정하지 않습니다.

## 7. 새 캐시 산출물

다음 옵셔널 parquet이 추가됩니다.

```text
versions/<cache_version>/aux_raw.parquet
versions/<cache_version>/credit_episode_nodes.parquet
versions/<cache_version>/credit_episodes.parquet
```

기존 캐시에 이 파일들이 없어도 UI는 깨지지 않습니다.

첫 v0.6.0 refresh 이후부터 새 산출물이 저장됩니다.

## 8. 확인지표 일시 실패 처리

기존에는 최신 방향 행을 과거 성공값으로 이어 썼습니다.

v0.6.0에서는 기업 신용 엔진의 과거 경로가 필요한 BBB·A·CP에 대해:

1. 이번 수집 실패
2. 과거 실제 정상 방향값 검색
3. 아직 stale이 아니면 마지막 실제 원자료 경로도 검색
4. 에피소드 엔진 입력 복구
5. 오래된 자료면 엔진에서 사용하지 않음

으로 처리합니다.

## 9. UI 변화

탭 8개 구조는 유지합니다.

### 현재 상황

- 핵심 6개
- 기업 신용 범위와 지속
- 기존 지표별 상세 설명

### 오늘의 해석

- 기업 신용 에피소드
- 3개 영역
- 조합 해석
- HY-BBB 상세 가이드
- 확인 지표 5개 상세 가이드
- 외부 참고 2개 상세 가이드

### 지난 30일 흐름

- `움직인 순서` 표현 제거
- `새 변화가 처음 확인된 날짜`로 변경
- 실제 선행·후행 판정이 아님을 명시
- 현재 기업 신용 범위·지속과 최근 상태 전환 표시

### 지표 설명

총 14개 선택 가능:

- 핵심 6
- HY-BBB 렌즈 1
- 확인 5
- 외부 참고 2

broad IG는 일반 선택 목록에서 제외됩니다.

2. HY-BBB 현재 차이와 약 1개월 변화
3. 확인 지표 5개

5. 기업 신용 범위·지속

## 12. 되돌리기

문제가 생기면 v0.5.1 코드로 되돌리고 다시 refresh할 수 있습니다.

v0.6.0의 새 parquet은 옵셔널 파일이므로 v0.5.1이 기존 핵심 아티팩트를 읽는 동작을 방해하지 않습니다.

다만 v0.6.0 이후의 사용자 화면·Telegram 해석은 v0.5.1과 구조가 다르므로, 버전 롤백 시 해당 기능은 사라집니다.
