# RiskRadar v0.3 → v0.4 마이그레이션 가이드북

이 문서는 v0.3에서 v0.4로 바뀐 내용을 전부 정리한다. (경제일정·이벤트 플레이북은 v0.4 범위에서 제외했다.)

---

## 1. 한눈 요약 — v0.4가 한 일

v0.3은 6개 지표를 **각각** 잘 보여주는 계기판이었다. v0.4는 그 위에 "지표를 **함께** 읽는 층"을 얹었다. 단, 종합 위험점수나 매수·매도 판단은 끝까지 만들지 않는다.

1. **보조지표 3개 추가** — 장기금리·신용 변화의 원인을 구분하는 확인용 지표.
2. **3축 복합 조망** — 6개를 3개 위험축으로 묶어 "지금 어느 축이 움직이나"를 본다.
3. **조건부 해석 엔진** — 관찰된 조합마다 "가능한 설명 → 확인지표로 지지/약화 → 충돌 → 불확실성"을 규칙 기반으로 생성.
4. **"오늘의 해석" UI 탭** — 위 세 가지를 한 화면에 노출.

---

## 2. 새로 생긴 핵심 개념 4가지

1. **보조지표(2층)** — Breakeven·IG OAS·Term Premium. 핵심 6개(1층)와 분리한다. 3축·종합상태에 **넣지 않고**, 원인 확인에만 쓴다. 출력은 방향(상승/하락/보합/판정불가)뿐. 셋 다 FRED 소스라 별도 파싱 의존이 없다.

2. **3축** — 변동성·신용(VIX+HY OAS) / 경기 사이클(T10Y3M) / 금리 방향(30Y+2Y+10Y Real). 6개를 그대로 더하면 금리를 네 번 세게 되므로, 축으로 묶어 중복 카운트를 없앤다. 변동성·신용은 VIX(빠른 센서)와 HY(확인 센서)를 **동등 투표하지 않는다**. 금리 방향은 **다수결하지 않는다**(엇갈림=혼합을 정보로 남긴다).

3. **조건부 해석** — 하나의 조합에 정답 하나를 강요하지 않는다. 확인지표가 여러 설명을 동시에 지지하면 "복합"으로 남기고, 판정불가 지표는 그 확인만 보류한다.

4. **근거 등급 / C등급 컷** — 상태·축·조합 판정에 쓰는 임계값은 대부분 학술 기준이 아니라 운영상 정한 C등급 값이다. 화면에 "내부 참고 규칙(C등급), 투자 신호 아님" disclaimer를 붙인다.

## 3. 단계별 변경

1. **1단계 — 보조지표 수집 + 방향 판정**
   - FRED에서 3개 수집(T10YIE / BAMLC0A0CM / THREEFYTP10).
   - 방향 = 최근 1개월(21거래일) 변화의 부호. 변화 크기가 자기 과거 |변화| 분포 대비 미미하면 보합으로 눌러 노이즈 차단(등속 추세를 보합으로 놓치지 않게 백분위-부호가 아닌 부호+크기유의성 방식).
   - refresh에 연결: `aux_signal_matrix.parquet`로 저장, freshness 표시, 실패해도 전체 안 깨뜨림.

2. **2단계 — 3축 복합 조망 엔진**
   - 변동성·신용 5개 상태(A~E). E(변동성 진정·신용 지속)는 최근 10개 창의 VIX 이력으로 C(신용 단독)와 구분.
   - 경기 사이클은 T10Y3M 경로 재사용, 금리 방향은 상승/하락/혼합/없음.
   - 최상단은 "3축 중 N축 변화"만. 단일 점수·행동 라벨 없음.

3. **3단계 — 조합 탐지 + 조건부 해석**
   - 조합 4개. 30Y↑·2Y↓는 완성형(확인지표 4개), 나머지 3개는 골격.
   - 지지/약화 집계 → 2개 이상 지지면 충돌 안내 → 불확실성.
   - refresh의 `data_quality`에 `axes`·`readings` 저장.

4. **4단계 — "오늘의 해석" UI 탭**
   - Market Stress Board 다음 2번째 탭.
   - 렌더링은 gradio 비의존 모듈(`today_view.py`)로 분리해 테스트 가능.

## 4. 파일별 변경 (v0.3 대비 전부)

**신규 소스 (`src/riskradar/`)**
- `version.py` — 버전 단일 소스(`__version__ = "0.4.0"`).
- `aux_config.py` — 보조지표 3개 정의 + 방향 판정 컷.
- `aux_indicators.py` — 보조지표 수집 + 방향(상승/하락/보합/판정불가).
- `axis_engine.py` — 3축 + 최상단 복합 조망.
- `combo_rules.py` — 조합 카탈로그(탐지 조건 + 확인지표 분기).
- `interpretation_engine.py` — 조건부 해석 생성.
- `today_view.py` — 오늘의 해석 마크다운 렌더(gradio 비의존).

**수정 소스**
- `fred_client.py` — 저수준 `fetch_fred_series` 노출(보조지표가 같은 경로 재사용). 기존 `fetch_all` 인터페이스 유지.
- `cache_store.py` — 옵셔널 아티팩트(`aux_signal_matrix`) 추가로 하위호환 보장, 두 스토어에 `load_data_quality()` 추가.
- `refresh_service.py` — 보조지표·3축·조건부 해석 통합, `aux_fetcher` 주입 지원. 보조/해석 실패는 전체를 안 깨뜨림.
- `ui.py` — "오늘의 해석" 탭 + data_quality/aux 로드.

**신규 테스트 (`tests/`)**
- `test_aux_v0_4.py` / `test_aux_integration_v0_4.py` / `test_axis_v0_4.py` / `test_interpretation_v0_4.py` / `test_today_view_v0_4.py`.

**기타**
- `pyproject.toml` — version `0.1.0` → `0.4.0`.
- `CHANGELOG.md` — 단계별 이력 신규.
- 제거: `axis_config.py`(어디서도 참조되지 않던 dead 파일).

## 5. 데이터 흐름 변화

v0.3: FRED 6개 → 상태 계산 → 캐시(parquet) → UI

v0.4 추가분:
1. FRED 보조 3개 → 방향 판정 → `aux_signal_matrix.parquet`
2. 6개 상태 → 3축 → 조합 탐지 → 조건부 해석
3. 축·조합 결과를 `data_quality.json`에 저장(`axes`, `readings`)
4. UI가 `load_data_quality()` + `aux_signal_matrix`를 읽어 "오늘의 해석" 렌더

핵심: 보조지표·해석은 **부가 계층**이라, 실패해도 핵심 6개 계기판은 그대로 동작한다.

---

## 6. 튜닝 대상 (C등급 초기값)

과거 국면(2008/2020/2022)으로 돌려보며 조정할 값들:
- 보조지표 방향: `lookback_obs=21`, `flat_abs_pct=40`, `min_obs=250`, `span_guard_days=45` (`aux_config.py`)
- 3축: `vix_persist_window=5`, `vix_persist_min=3`, `e_link_window=10` (`axis_engine.py`)

이 값들은 철학처럼 고정하지 말고, 노이즈가 얼마나 뜨는지 보고 조정한다.

## 7. 테스트

- 신규 39(방향 7 + aux통합 4 + 축 15 + 해석 7 + 오늘의해석 6) + 기존 28 = **67개 통과**.
- 철학 가드 테스트 포함: 축·해석·UI 출력에 "위험/안전/조심/매수하/매도하" 등 행동 지시어가 없는지, 종합 점수 필드(`score`, `risk_level`)가 없는지 검증.

## 8. 적용 방법

1. `riskradar_v0_4_0_full` 전체를 repo에 반영(또는 v0.3 위에 덮어쓰기).
2. `pyproject.toml` version은 이미 `0.4.0`.
3. GitHub에 push → sync 워크플로우가 HF Space로 자동 복제.
4. 다음 배치(cron-job.org 트리거) 후 "오늘의 해석" 탭에 축 조망·조합 해석·보조지표 방향이 표시된다.
5. 첫 배치 전이거나 옛 캐시면 해당 탭은 placeholder를 보여준다(대시보드는 정상 동작).
