# RiskRadar v0.8.2 — Codex One-File Handoff

## 사용 방법

Codex에서 `codex-riskradar-work` 브랜치를 연 뒤 다음 한 줄만 입력한다.

> `CODEX_HANDOFF_v0_8_2.md를 처음부터 끝까지 읽고 명세대로 v0.8.2를 구현해. 충돌이 없으면 바로 진행하고, 완료 후 테스트 결과와 변경 파일을 보고해.`

이 문서는 v0.8.2 UI·표현 개편의 확정 구현 계약서다. 제품 설계는 이미 끝났다. 새로운 화면 구조·지표·점수·임계값·시각화·기능을 임의로 제안하거나 추가하지 않는다.

| 탭 | 역할 |

# 4. 공통 UI 원칙

## 카드 정보 순서

1. 약칭
2. 쉬운 한국어 설명
3. 현재값
4. 현재 상태
5. 최근 약 1개월 변화
6. 기존 percentile이 있을 때만 최근 자료 범위 내 위치

약칭 다음에 한국어 설명이 나오는 기존 방식을 유지한다.

## 개념 분리

- state ≠ trend ≠ duration
- HY core level ≠ credit episode state
- HY−BBB는 독립 시장이 아니라 관계 lens
- OAS 상승 ≠ 회사채 절대금리 반드시 상승
- Term Premium은 장기금리 요인이지 30Y 구성에 더하는 항목이 아님

## 단위

- 기존 `%`, `%p` 유지
- UI 전체를 bp로 전환하지 않음
- `1개월`보다 `최근 약 1개월` 또는 `약 1개월`

## 색과 가독성

- 상태색·관계색·방향색을 구분
- 상승을 무조건 빨강, 하락을 무조건 초록으로 칠하지 않음
- 정상·조용한 카드도 비교 대상이므로 카드 전체 opacity를 낮추지 않음
- 숫자와 텍스트는 항상 선명하게, 배경과 테두리만 차분하게
- 상태 의미를 색만으로 전달하지 않음

## 위치 표시

- 위험 확률처럼 보이지 않게 함
- 새 percentile 계산 금지
- 기존 artifact만 사용
- 자료 없으면 `비교 불가`
- ICE 회사채 3년 위치를 장기 역사 희귀도로 표현하지 않음

## 반응형

- 모바일 가로 스크롤 최소화
- 대형 현황 카드: 모바일 1열
- 미니 카드: 모바일 2열, 매우 좁으면 1열
- 상세 판정 근거는 모바일 세로 레코드 권장

# 6. 현황 탭

## 최종 순서

```text
자료 기준일 · 데이터 상태

1. 현황 한눈에
   [기업 신용] [장기금리] [시장 변동성]

2. 시장 해석

3. 주요 지표

4. 최근 상태 변화

5. 판정 근거
```

기존 첫 탭의 다음 블록은 그대로 유지하지 않는다.

- 기존 domain strip
- `새로 달라진 점` 중심 구조
- 독립 `현재 추세`
- 독립 신용 2×2 중복
- 독립 `핵심 6개`
- 독립 `다음에 볼 것`
- 중복 장문 `왜 이렇게 봤나`

필요한 정보는 새 구조에 흡수한다.

## 자료 기준일·데이터 상태

정상일 때 작고 간결하게:

```text
2026-07-10 시장 데이터 기준 · 정상 갱신
```

시장 관측일과 Dataset 재조회 시각을 혼동하지 않는다. stale·failed·carried-forward·recovery gap일 때만 큰 경고로 확장한다. 관리·진단은 기존 접힘을 유지한다.

## 현황 한눈에 대형 카드 3개

### 기업 신용

- 대표값: HY OAS
- 상태: credit episode의 HY node 상태
- 지속: 정확한 confirmed_at이 있고 `상승 확인` 또는 `높은 수준 지속`일 때만
- 관계: 현재 신용 범위와 HY−BBB 기존 lens 사용
- 새 확산 단계 금지
- 기존 HY percentile과 약 1개월 변화 사용

### 장기금리

- 대표값: 30Y
- 기존 30Y core state
- 기존 same-maturity 30Y composition으로 실질 30Y와 30Y 일반·물가연동 금리차 관계 표시
- 같은 방향 또는 일부 상쇄 표현
- Term Premium을 구성요인으로 더하지 않음

### 시장 변동성

- 대표값: VIX
- 기존 VIX state와 변화값 사용
- 새 `안정` 종합 판정 금지

## 시장 해석

숫자를 반복하지 않고 관계만 설명한다.

- 전체 흐름 최대 2문장
- 의미 최대 2개
- 확인할 점 최대 1개

직접 관계는 직접 말하고, 다른 지표에 미치는 영향은 `요인`, `압력`으로 표현한다. 특별한 관계가 없으면 억지 해석을 만들지 않는다.

## 주요 지표 미니 카드

### 기업 신용

- HY
- BBB
- A
- CP

### 금리

- 30Y
- 2Y
- 실질 10Y
- 10Y-3M
- 10Y Term Premium

VIX는 대형 카드에만 두고 반복하지 않는다.

보조 지표에 새 상태를 만들지 않는다. 기존 방향 결과만 `최근 상승`, `최근 하락`, `뚜렷한 움직임 없음`, `확인 불가`로 표시한다.

## 최근 상태 변화

- permanent decision ledger 또는 권위 있는 저장 decision history만 사용
- raw data를 현재 규칙으로 재계산해 사건을 만들지 않음
- 실제 상태 전환만
- 최근 30일
- 최대 5건

빈 상태:

```text
최근 30일 동안 확인된 상태 변화가 없습니다.
```

기록 부족:

```text
상태 변화 기록을 쌓는 중입니다.
```

현재 UI 경로로 ledger를 읽을 수 없다면 임의 fallback을 만들지 말고 수정 전에 보고한다.

## 판정 근거

기본 접힘 `판정 근거 자세히 보기`.

기존 state reason과 저장 판정만 사용한다. 새 설명 로직 금지.

# 7. 신용 탭

큰 구조는 유지한다.

```text
기업 신용 2×2
현재 신용 흐름
HY−BBB 관계
최근 90일 신용 변화
지난 신용 변화
현재 신용 상태 자세히
지표 뜻과 읽는 법
```

## 카드

- 약칭 먼저
- 한국어 설명 다음
- 현재값
- credit episode 상태
- 최근 약 1개월 변화

subtitle:

- HY: 낮은 등급 회사채
- BBB: 투자등급 경계
- A: 우량 회사채
- CP: 단기 기업자금 금리차

제목 아래:

```text
회사채 수치는 비슷한 만기의 국채 대비 추가 금리입니다.
```

quiet 카드 전체 opacity 제거.

## 현재 신용 흐름

변화가 없을 때 문구를 정확히 유지한다.

```text
현재 기업 신용에 새로 강조할 움직임이 없습니다.
```

## HY−BBB

```text
HY−BBB
HY와 BBB의 추가 금리 차이
```

관계 문구:

```text
HY와 BBB의 추가 금리 차이가 크게 변하지 않았습니다.
HY가 BBB보다 더 올라 차이가 벌어졌습니다.
BBB가 HY보다 더 올라 차이가 줄었습니다.
두 시장의 차이를 현재 판단하기 어렵습니다.
```

## 최근 90일

- 제목 `최근 90일 신용 변화`
- 기본 최대 10건
- 나머지는 `+ N개 변화가 더 있습니다`
- `이전 고점 돌파` → `추가 상승`
- 한 줄 안내 `확인된 상태 변화만 표시합니다.`

## 지난 변화

아코디언 제목 `지난 신용 변화`.

첫 줄:

```text
현재 제공되는 공식 자료로 다시 계산한 기록입니다.
```

permanent ledger 기록처럼 표현하지 않는다.

## 상세

- `신용 변화 상세` → `현재 신용 상태 자세히`
- `실제 시장별 현재 상태` → `신용시장별 현재 상태`
- `기준선 대비 +0.25%p 남음` → `상승 전 기준보다 0.25%p 높은 상태`

## 설명

바깥 아코디언 `지표 뜻과 읽는 법`.

# 8. 금리 탭

## 구조

```text
1. 금리 현황
   [30Y] [2Y]
   [실질 10Y] [10Y-3M]

2. 단기·장기 금리 관계

3. 30Y 금리 변화 나눠보기

4. 장기금리 참고
   [10Y Breakeven] [10Y Term Premium]

5. 지표 뜻과 읽는 법
```

## 금리 현황

약칭 먼저, 한국어 설명 다음.

- 30Y / 미국 30년 국채금리
- 2Y / 미국 2년 국채금리
- 실질 10Y / 물가 영향을 뺀 10년 금리
- 10Y-3M / 10년·3개월 국채금리 차이

기존 상태 문구 유지.

## 단기·장기 금리 관계

- `금리곡선` → `단기·장기 금리 관계`
- 2Y와 30Y의 최근 약 1개월 변화 비교
- 쉬운 한국어 요약
- 기본 화면의 bear/bull steepening/flattening 배지 제거
- 전문용어는 설명 탭 참고 수준만 허용

## 30Y 금리 변화 나눠보기

- `30년 금리` → `30Y 금리 변화 나눠보기`
- 상단 현황과 중복되는 현재 30Y 값 제거
- 유지: 전체 변화, 실질 30Y 변화, 30Y 일반·물가연동 금리차 변화, 상쇄·동방향 요약
- 기존 막대·반대 방향 숫자 배치 유지

표시명:

```text
실질 30Y
물가 영향을 뺀 30년 금리
```

```text
30Y 국채 금리 차이
일반·물가연동 30년 국채의 차이
```

기존 1개월·3개월 표는 아코디언 `약 3개월까지 비교` 안으로 이동한다.

## 장기금리 참고

실질 10Y 중복 카드를 제거하고 다음 두 카드:

```text
10Y Breakeven
일반·물가연동 국채금리 차이
```

```text
10Y Term Premium
장기채 추가 보상 추정치
```

안내:

```text
아래 지표는 30Y 금리 구성에 직접 더하는 값이 아니라 별도로 함께 보는 참고 자료입니다.
```

## 설명

기존 `금리 지표 설명`과 `금리 설명·주의사항`을 바깥 아코디언 `지표 뜻과 읽는 법` 하나로 통합한다.

반드시 남길 주의:

- 30Y와 실질 30Y는 같은 만기
- 일반·물가연동 금리차는 순수 물가 기대만 뜻하지 않음
- Term Premium은 모형 추정치
- Term Premium은 30Y 구성에 더하는 항목이 아님

# 9. 흐름 탭

시간 경로에만 집중한다.

## 최종 구조

```text
1. 지난 30일 요약
   - 한 달 사이 크게 달라진 지표 최대 3개
   - 크게 움직였다가 되돌아온 지표
   - 현재까지 남아 있는 흐름

2. 지표별 흐름
   [지표 선택]
   지난 30일 차트 기본 노출

   ▸ 현재 제공되는 전체 기간 흐름
   ▸ 지난 30일 상세 표
   ▸ 지표 뜻과 읽는 법
```

유지:

- 한 달 사이 어떻게 달라졌나
- 되돌림
- 현재 남아 있는 추세
- 지표 선택
- 지난 30일 차트
- 전체 자료 차트
- 지난 30일 상세 표
- 선택 지표 설명

메인에서 제거:

- 별도 `2년 금리와 30년 금리는 어떻게 움직였나`
- `기업 신용 변화`
- `지금부터 확인할 것`
- 재구성 판정 날짜의 메인 노출

차트 배치:

- 30일 차트 기본
- 전체 기간 차트 접힘
- 상세 표 접힘
- 설명 접힘

`전체 원자료 흐름` 대신 `현재 제공되는 전체 기간 흐름`.

# 10. 비교 탭

표 구조를 유지한다.

## 최신 지표 비교


- NFCI·STLFSI 별도 중복 아코디언 제거
- 각 설명 안에서 외부 참고 지표이고 핵심 상태를 직접 바꾸지 않는다고 명시

- 오늘 → 현황
- 금리곡선 → 단기·장기 금리 관계
- 30년 금리 → 30Y 금리 변화 나눠보기

권장:

```text
HY도 상승 → 낮은 등급 기업이 국채보다 추가로 부담하는 금리가 커집니다.
BBB·A까지 상승 → 투자등급 기업의 국채 대비 추가 조달비용까지 커집니다.

```
