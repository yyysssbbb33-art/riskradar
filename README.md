# RiskRadar v0.6.0 — 미국 시장 흐름 신호판

RiskRadar는 미국 시장의 **주식시장 흔들림·기업 신용 부담·경기 흐름·금리 움직임**을 따로 보고, 서로 같이 움직이는 범위와 아직 남아 있는 변화를 설명하는 읽기 전용 대시보드입니다.

단일 위험점수, `안전/위험` 최상위 라벨, 매수·매도 권고, 근거 없는 침체확률을 만들지 않습니다. 결과가 엇갈리면 하나의 결론으로 억지로 합치지 않습니다.

## v0.6.0 구조

```text
1층 — 핵심 신호
VIX · HY OAS · 10년-3개월 금리차 · 30년 금리 · 2년 금리 · 실질 10년 금리

기업 신용 해석 렌즈
HY-BBB 등급 차이

2층 — 확인 지표
물가 예상 · 장기채 추가 보상 · BBB OAS · A OAS · CP 등급 간 금리 차이

3층 — 외부 참고
NFCI · STLFSI
```

### 핵심 6개

- **주식시장이 예상하는 흔들림(VIX)** — 주식시장 예상 변동성
- **신용등급 낮은 기업의 추가금리(HY OAS)** — 위험도가 높은 기업의 회사채 부담 자체
- **10년 금리와 3개월 금리의 차이** — 경기보다 먼저 움직이는 금리 관계
- **미국 30년 국채금리** — 장기금리
- **미국 2년 국채금리** — 가까운 기준금리 예상에 민감한 금리
- **물가 영향을 뺀 10년 금리** — 물가 효과를 제외한 장기 금리 부담

### 기업 신용 렌즈: HY-BBB

`HY OAS - BBB OAS`는 별도 시장이 아닙니다. HY와 BBB를 다시 조합한 파생값이므로 기업 신용 범위 엔진의 독립 노드로 세지 않습니다.

이 렌즈는 다음 질문에만 답합니다.

> 현재 부담 증가가 투기등급 기업에 특히 집중되고 있는가, 아니면 투자등급 경계 기업도 함께 움직이는가?

### 확인 지표 5개

- **장기금리가 왜 움직이나** — 채권시장이 보는 10년 물가 예상, 장기채 추가 보상
- **기업 부담이 어디까지 번졌나** — 투자등급 경계 기업의 추가금리(BBB), A등급 기업의 추가금리(A)
- **단기 기업자금시장도 영향을 받나** — 30일 A2/P2 비금융 CP 금리 - 30일 AA 비금융 CP 금리

### 외부 참고 2개

- **미국 금융시장 전반의 자금 사정(NFCI)**
- **미국 금융시장 전반의 불안(STLFSI)**

두 지표는 외부 공식 종합지표를 참고하는 용도입니다. 핵심 상태, 3개 영역 개수, 기업 신용 에피소드 참여, 조합의 지지·약화 판정에 넣지 않습니다.

## 기업 신용 범위·지속 엔진

v0.6.0의 가장 큰 변화입니다. 순서를 억지로 주장하는 `선행/후행 엔진` 대신 다음을 먼저 봅니다.

1. 현재 기업 신용 에피소드가 있는가
2. 실제 시장 중 어디가 참여하고 있는가
3. 각 시장의 변화가 새로 시작됐는가, 이어지는가, 되돌아오는가
4. 처음 벌어진 변화가 아직 얼마나 남아 있는가
5. 변화가 저신용 기업에 집중됐는가

### 실제 시장 노드

```text
회사채 시장
HY → BBB → A

다른 자금시장
CP
```

- **HY** — 위험도가 높은 기업의 회사채
- **BBB** — 투자등급 경계 기업의 회사채
- **A** — 투자등급 안쪽의 A등급 회사채
- **CP** — 단기 기업자금시장

과거 broad IG(`BAMLC0A0CM`)는 BBB를 포함하므로 BBB 뒤의 독립 확산 단계로 사용하지 않습니다. 구버전 캐시 호환과 운영 진단을 위해 수집 경로만 유지하고, 범위 엔진과 일반 사용자 상세 목록에서는 제외합니다.

### 노드 상태

- 평소 상태
- 초기 변화
- 새로 상승
- 상승 지속
- 되돌림
- 정상화

`초기 변화`는 관찰 단계이며 공식 참여 시장으로 세지 않습니다.

### 에피소드 상태

- **활성** — 최근 새 참여나 의미 있는 확대·상태 전환이 있음
- **휴면** — 과거 변화의 잔존은 남아 있지만 최근 의미 있는 활동은 없음
- **종료** — 참여 시장의 변화가 동결된 사전 기준선으로 충분히 되돌아옴

휴면 뒤 새 변화가 생기면 새 에피소드로 시작하고, 이전 에피소드의 잔존 변화가 있으면 별도 주석으로 남깁니다.

### 변화 시작과 잔존을 분리

```text
변화 강도
→ 후보 발생 · 추정 시작 · 확정

동결된 사전 기준선 대비 잔존 변화
→ 상승 지속 · 되돌림 · 정상화
```

오르는 속도가 멈췄다고 문제가 끝난 것으로 보지 않습니다.

확정 시 다음 값을 동결합니다.

- 추정 시작일
- 확정일
- 시작 직전 기준선

기준선은 추정 시작일 **직전** 유효 관측치들의 중앙값으로 잡아 초기 상승분이 섞이지 않게 합니다. 확정 뒤에는 소급 수정하지 않아 Telegram과 UI의 과거 설명이 뒤집히지 않게 합니다.

### 순서 표현 정책

v0.6.0은 사용자에게 실제 선행·후행을 주장하지 않습니다. 관련 타임스탬프는 처음부터 저장하지만 1차 산출물은 **범위·지속·잔존**입니다.

- `주식시장이 먼저 움직임` → **주식시장 쪽만 움직임**
- 지난 30일의 날짜 목록 → **새 변화가 처음 확인된 날짜**
- 날짜가 앞섰다는 사실만으로 원인·선행 관계를 만들지 않음

VIX는 신용 에피소드 노드가 아니라 `에피소드 시작 무렵 주식시장 불안도 있었는가`를 보는 맥락 정보입니다.

## ICE BofA 회사채 자료 정책

2026년 4월 이후 FRED에서 ICE BofA 회사채 계열은 최근 약 3년 자료만 제공합니다. v0.6.0은 이 현실을 숨기지 않습니다.

- HY는 **최근 3년 중 현재 위치**만 표시
- 실제 5년·10년 달력 범위가 없으면 5년·10년 위치를 계산하지 않음
- BBB·A 상세 화면은 `현재 확보된 자료 범위 안 위치`라고 표시
- 최근 3년을 `장기 역사`, `역사적으로 이례적`, `과거 위기 수준`이라고 부르지 않음
- 회사채 범위·지속 엔진은 최근 공식 자료 범위에서 현재 변화·지속·확산을 봄
- 장기 경제학적 의미와 과거 사례는 논문·공식 연구·사례 연구의 역할로 분리

HY의 기존 5년·10년 위치가 관측치 개수만으로 계산되던 문제도 수정했습니다. 이제 실제 달력 커버리지가 충분해야 해당 기간 백분위를 계산합니다.

## 확인지표·외부 참고 상세 가이드

지표 설명 탭에서 다음 14개 설명을 볼 수 있습니다.

- 핵심 6개
- HY-BBB 렌즈 1개
- 확인 지표 5개
- 외부 참고 2개

각 확인지표와 외부 참고는 다음 8칸 구조를 유지합니다.

1. 뭘 측정하나
2. 뭘 우선 보나
3. 값 구간 감각
4. 같이 볼 지표 + 왜
5. 흔한 오해
6. 오를 때 / 내릴 때 방향
7. 근거 수준
8. 근거의 한계

현재 데이터 화면에서는 `현재값 → 약 1개월 변화 → 지금 해석 → 같이 볼 지표의 현재 결과 → 결과별 해석 → 8칸 설명` 순서로 연결합니다.

## 지난 30일 흐름

핵심 6개의 과거 point-in-time 경로를 현재 규칙으로 재구성합니다.

- 한 달 사이 어떻게 달라졌나
- 한때 크게 움직였다가 되돌아온 변화
- 아직 남아 있는 변화
- 새 변화가 처음 확인된 날짜
- 2년·30년 금리의 엇갈림
- 현재 기업 신용 범위·지속 상태

엔진 내부는 최대 90개 관측치를 사용하지만 사용자 월간 화면은 최근 30일을 보여줍니다.

## Telegram

정상·부분성공 알림은 **결과를 먼저 전부 보여준 뒤 해석**합니다.

1. 핵심 6개 수치: 최신값 · 약 1개월 · 약 3개월 · 상태
2. HY-BBB 렌즈: 현재 차이 · 약 1개월 변화 · 집중도 해석
3. 확인 지표 5개: 최신값 · 약 1개월 · 방향
4. 외부 참고 2개: 최신값 · 약 1개월 · 현재 위치
5. 기업 신용 범위·지속: 에피소드 상태 · 범위 · 노드 상태 · VIX 맥락
6. 3개 영역
7. 눈에 띄는 조합과 현재 해석
8. 외부 참고 요약

정상 알림에는 캐시 버전·배치 시각 같은 운영 메타데이터를 넣지 않습니다.

5. **전체 지표 비교**

## 데이터 파이프라인

```text
cron-job.org
  → GitHub workflow_dispatch
  → run_refresh.py
  → FRED 핵심 6개 + 확인/외부/호환 지표 수집
  → 상태·3개 영역·조합·기업 신용 범위·지속 계산
  → HF Dataset versioned cache 저장
  → Telegram 알림

HF Space
  → HF Dataset의 최신 active cache를 읽기
  → FRED 직접 수집 없이 화면 표시
```

새 옵셔널 산출물:

```text
aux_raw.parquet
credit_episode_nodes.parquet
credit_episodes.parquet
```

옛 캐시에 새 파일이 없어도 UI 전체가 깨지지 않습니다. 모든 부속 파일은 하나의 `active_cache_version`에 고정해 읽습니다.

확인지표가 일시 실패하면 직전 실제 성공값을 이어 쓰되 최신성을 다시 계산합니다. 기업 신용 엔진에 필요한 BBB·A·CP 원자료도 직전 실제 원자료가 아직 오래되지 않았다면 복구해 경로를 유지합니다. 오래된 원자료는 현재 엔진 입력으로 사용하지 않습니다.

## 검증 도구

새 엔진의 임계값을 몇 개 위기 사례에 맞춰 튜닝하지 않습니다. 개발용 검증은 다음 세 축입니다.

- 평온기 오탐률
- 알려진 스트레스 국면의 상식적 해석
- 임계 백분위 ±2~3 변화에 대한 민감도

민감도는 에피소드 집합, 참여 시장, 시작일, 지속시간이 작은 임계값 변화에 과도하게 뒤집히지 않는지 확인하는 용도입니다.
