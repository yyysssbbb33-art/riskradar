# RiskRadar v0.4.6 — 미국 시장 흐름 신호판

미국 시장의 **주식시장 흔들림·기업 자금 부담·경기 흐름·금리 움직임**을 따로 보고, 함께 읽는 대시보드입니다.
단일 위험점수, 매수·매도 신호, 침체 확률을 만들지 않습니다.

## 핵심 구조

```text
핵심 6개 지표
주식시장이 예상하는 흔들림(VIX)
신용등급 낮은 기업의 추가금리
10년 금리와 3개월 금리의 차이
30년 국채금리 · 2년 국채금리
물가 영향을 뺀 10년 금리

        ↓

3개 영역으로 함께 보기
주식시장 흔들림·기업 자금 부담 / 경기 흐름 / 금리 움직임

        ↓

결과에 따라 달라지는 해석
지금 보이는 모습 → 가능한 설명 → 같이 볼 지표 → 현재 결과 → 결과가 달라질 때의 해석
```

원인을 더 나눠 볼 때만 아래 3개 확인지표를 사용합니다. 이 지표들은 3개 영역 개수나 점수에 더하지 않습니다.

- 채권시장이 보는 10년 물가 예상 (`T10YIE`)
- 신용등급 높은 기업의 추가금리 (`BAMLC0A0CM`)
- 장기채 추가 보상, Kim-Wright 모델 (`THREEFYTP10`)

## v0.4.6 주요 변경

- 화면 전체를 다시 훑어 직역체와 시장 전문용어를 실제로 무슨 일이 일어나는지 설명하는 말로 교체
- 10년 금리와 3개월 금리의 상태를 `역전/재정상화` 대신 실제 금리 관계로 표시
- 핵심 6개 지표의 `다음으로 볼 것` 문장을 쉬운 한국어로 전면 재작성
- `지표를 같이 보는 법` 가이드도 같은 원칙으로 전면 재작성
- `최근 5년 중 현재 위치`는 위험확률이 아니라 과거 순위라는 점이 바로 보이도록 표시
  - 예: `높은 편 · 82%의 날보다 높음`
- 오래된 캐시와 내부 코드명은 그대로 호환하되 사용자 화면에서만 쉬운 표현 사용

## 화면 탭

1. **현재 상황** — 핵심 6개 최신값·상태·지금 뜻
2. **오늘의 해석** — 여러 지표를 같이 봤을 때 눈에 띄는 모습과 결과별 해석
3. **최근 30일** — 지표별 흐름 + 상세 설명
4. **같은 날짜 비교** — 모든 지표에 실제 데이터가 함께 있는 날짜 기준 비교
5. **전체 지표 비교** — 최신값·최근 변화·과거 위치·표시 이유
6. **차트** — 원자료 흐름
7. **데이터 상태** — 운영 점검
8. **지표 설명** — 개별 지표 설명 + 지표를 같이 보는 법

## 화면 용어

| 화면 | 내부 계산 | 쉬운 뜻 |
|---|---|---|
| 약 1개월 변화 | 20개 관측치 전 대비 | 대략 한 달 전보다 얼마나 변했는지 |
| 약 3개월 변화 | 60개 관측치 전 대비 | 대략 세 달 전보다 얼마나 변했는지 |
| 최근 5년 중 현재 위치 | 최근 5년 안의 상대 위치 | `높은 편 · 82%의 날보다 높음`처럼 표시 |
| 최근 10년 중 현재 위치 | 최근 10년 안의 상대 위치 | 장기적으로 지금 값이 높은지 낮은지 |

`82%의 날보다 높음`은 위험확률 82%가 아닙니다. 최근 기간의 관측일을 줄 세웠을 때 현재 값보다 낮았던 날이 82%였다는 뜻입니다.

## 오늘의 해석 원칙

- 하나의 지표나 조합에 정답 하나를 강요하지 않음
- `같이 볼 지표`는 이름만 알려주지 않고 오를 때·내릴 때·큰 움직임이 없을 때 해석을 각각 설명
- 여러 설명이 동시에 맞을 수 있으면 하나로 억지로 합치지 않음
- 오래됐거나 확인할 수 없는 자료는 현재 해석 근거에서 제외
- `위험/안전/매수/매도` 같은 행동판단을 만들지 않음

## 데이터 파이프라인

```text
외부 스케줄러 / GitHub workflow_dispatch
  → run_refresh.py
  → FRED 핵심 6개 + 보조 3개 수집
  → 상태·3개 영역·조합·조건부 해석 계산
  → HF Dataset versioned cache 저장
  → Telegram 알림

HF Space
  → 캐시만 읽어서 화면 표시
```

## 데이터 원칙

- raw 관측값만 계산 입력에 사용
- 상태는 매 refresh마다 raw에서 전체 재계산
- 백분위·변화·상태는 point-in-time
- Synced Snapshot은 forward fill 이전 실제 원본 관측일 교집합 사용
- 보조지표 실패가 핵심 6개 대시보드를 깨뜨리지 않음
- 같은 날짜 여러 스냅샷은 원본 버전에 보존하되 히스토리 UI는 마지막 한 개만 표시

## 핵심 원칙

> RiskRadar는 위험을 대신 판정하지 않습니다.
>
> 현재 어떤 지표와 영역이 움직이는지, 어떤 설명이 가능하며 어떤 추가 데이터가 그 설명을 지지하거나 약화하는지 보여줍니다.
>
> 결과가 엇갈리면 억지로 하나의 결론을 만들지 않습니다.
