# RiskRadar v0.8.3 — 미국 시장 흐름 신호판

RiskRadar는 미국 시장의 **주식시장 변동성·기업 신용·경기 흐름·금리 움직임**을 따로 보고, 서로 같이 움직이는 범위와 아직 남아 있는 변화를 설명하는 읽기 전용 대시보드입니다.

단일 위험점수, `안전/위험` 최상위 라벨, 매수·매도 권고, 근거 없는 침체확률을 만들지 않습니다. 결과가 엇갈리면 하나의 결론으로 억지로 합치지 않습니다.

## v0.8.3 릴리스 일관성 핫픽스

v0.8.3은 v0.8.2의 화면 구조와 판정 로직을 유지하면서, 릴리스 과정에서 빠진 문서·표현·호환 등록을 바로잡고 같은 누락이 다시 병합되지 않도록 자동 검사를 추가합니다.

- README를 현재 `현황 / 신용 / 금리 / 흐름 / 비교 / 설명` 구조와 v0.8.3 기준으로 갱신
- OAS를 회사채의 절대 금리나 실제 조달비용과 동일시하지 않고 **같은 만기 국채 대비 추가 금리**로 표현
- `금리곡선` 대신 기본 화면과 안내에서 **단기·장기 금리 관계**를 우선 사용
- v0.8.2·v0.8.3 UI가 직전 호환 캐시를 읽도록 데이터 버전 호환 목록 보완
- 버전·README·호환 목록·임시 전달 파일·금지 문구를 검사하는 릴리스 일관성 테스트 추가
- main 대상 PR에서 전체 pytest와 Gradio `Blocks` 생성 smoke를 자동 실행하는 GitHub Actions 추가

수집, threshold, 핵심 상태 규칙, 신용 에피소드 전이, 30Y 계산, 캐시 발행, decision snapshot·diff·ledger, prune, Telegram 조건은 변경하지 않습니다.

## v0.8.2 현황 중심 UI

v0.8.2는 새 시장 신호나 새 계산을 추가하지 않고, 핵심 상태를 더 빨리 읽도록 화면의 정보 순서와 한국어 표현을 정리했습니다.

- 첫 탭을 `오늘`에서 **`현황`**으로 바꾸고 `자료 기준일 → 현황 한눈에 → 시장 해석 → 주요 지표 → 최근 상태 변화 → 판정 근거` 순서로 재구성
- 기업 신용·장기금리·시장 변동성을 대형 카드로 배치하고 정상 카드도 선명하게 표시
- 신용 탭은 HY·BBB·A·CP 2×2 구조와 최근 90일 변화 기록을 유지하되 중복 설명을 축소
- 금리 탭을 `금리 현황 → 단기·장기 금리 관계 → 30Y 금리 변화 나눠보기 → 장기금리 참고` 순서로 정리
- 흐름 탭은 지난 30일의 시간 경로와 선택 지표 차트에 집중
- 비교 탭은 최신 관측일 차이를 안내하고 `같은 날짜로 비교`를 명확히 표시
- 설명 탭은 지표 선택기를 먼저 배치하고 상세 설명의 중복을 줄임

## v0.8.1 표현·디자인 정리

v0.8.1은 새 지표나 새 판정을 추가하지 않습니다. v0.8.0의 숫자 중심 구조를 유지하면서 **사용자 문구를 한곳에서 관리하고, 상세 설명을 실제 시장 결과 중심으로 다시 쓰며, 카드·표·아코디언에 절제된 색과 시각 계층을 추가**합니다.

- CP가 `하락 전환`인데 범위 요약에서 `상승 신호`라고 부르던 상태 요약 오류 수정
- 핵심 HY 카드는 상승 과정이 아니라 현재 수준을 보여주도록 `낮은 수준 / 중간 수준 / 높은 수준 / 매우 높은 수준`으로 분리
- 사용자 문구 단일 계층 `user_copy.py` 추가. 웹 상세·설명 탭·상태 가이드·조합 해석이 같은 결과 문구를 재사용
- `결과가 달라지면`, `잘 맞습니다`, `설명을 지지합니다`, `부담이 넓어집니다` 같은 추상 표현을 줄이고 실제 금리·스프레드·조달비용 결과를 먼저 설명
- 표현 강도 원칙을 고정: 직접 관계는 직접 말하고, 다른 시장지표 영향은 `끌어올리는 요인 / 낮추는 요인`, 경기·투자 같은 2차 결과만 조건부로 표현
- Term Premium은 `상승하면 10년·30년 장기금리를 끌어올리는 요인`으로 설명하고 모형 추정치라는 한계는 별도 참고로 분리
- BBB·A는 국채 대비 추가 금리와 추가 조달비용, CP는 신용도별 단기 조달금리 차이가 어떻게 달라지는지 직접 설명
- 신용 탭에는 HY−BBB·BBB·A·CP만, 금리 탭에는 30Y·2Y·실질 10Y·10Y-3M·일반/물가연동 국채 금리 차이·Term Premium을 배치
- NFCI·STLFSI는 설명 탭의 `시장 전체 참고 지표`로 이동
- 상세 설명을 `지금 데이터 → 지금 해석 → 같이 볼 지표 → 움직임별 결과 → 참고` 순서로 단순화하고 중복된 상세 표 제거
- Telegram도 `평소 범위 / 오르는 중` 대신 웹과 같은 `중간 구간 / 상승 / 하락` 언어를 사용
- 상태별 옅은 배경·테두리, 신용/금리/변동성 영역색, 숫자 우선 타이포그래피, 표·아코디언 스타일을 추가

수집, threshold, 상태머신 판정, 30년 금리 계산, decision snapshot·ledger, prune, Telegram 전송 로직은 v0.8.0과 같습니다.

## v0.8.0 수치 중심 대시보드

v0.8.0은 새 시장 신호를 추가하지 않습니다. v0.7.4에서 시작한 Scan-first UI를 더 밀어붙여, **상태만 말로 보여주던 화면에 숫자를 되돌리고 상태와 추세를 분리**합니다.

- 상단 `신용 / 금리 / 변동성` 카드에 **현재값 + 상태 + 1개월 변화** 표시
- `평소`라는 공통 상태명 제거. 지표별 실제 경계 기준에 맞춰 `경계 신호 없음`, `급격한 상승 없음`, `특이 신호 없음` 등으로 구분
- 기업 신용 상태를 `특이 신호 없음 / 상승 조짐 / 상승 확인 / 높은 수준 지속 / 하락 전환 / 신호 해제`로 통일
- `계속 이어지는 변화`를 **`현재 추세`**로 변경하고 `지속 중` 같은 중복 문구 제거
- HY·BBB·A·CP 2×2에 현재 수치와 1개월 변화를 표시하고 HY−BBB에도 현재 차이와 1개월 변화를 표시
- **금리 전용 탭** 추가: 금리 현황, 금리곡선, 30년 금리 구성, 장기금리 참고 지표를 한곳에 배치
- `오늘` 탭의 30년 금리 대형 패널 제거. 30Y는 핵심 카드에서 보고 상세는 금리 탭에서 확인
- 줄글형 지표 나열을 숫자 카드·3열 이하 표로 전환
- 비교 탭을 `최신 지표 비교`와 **사용자 선택형 `날짜별 지표 보기`**로 재구성

기본 카드 문법은 다음과 같습니다.

```text
현재값
상태
최근 1개월 추세
```

`상태`는 RiskRadar의 경계 기준에 걸렸는지를, `최근 추세`는 실제 1개월 방향과 변화량을 뜻합니다. 그래서 `급격한 상승 없음`이면서 1개월에는 소폭 상승할 수 있습니다.

판정 규칙, threshold, 데이터 수집, decision snapshot·ledger, Telegram 전송은 v0.7.4와 같습니다.

## v0.7.3 권위 있는 판정 기록 원장

v0.7.3은 새 시장 신호나 새 점수를 만들지 않습니다. v0.6.2부터 각 refresh가 실제로 저장해 온 `decision_snapshot.json` 중 **권위 있는 기록만** `audit/decision_ledger.parquet`에 장기 보존합니다.

- 현재 보존 중인 `versions/<cache_version>/decision_snapshot.json`을 훑어 `authoritative == true`인 기록만 초기 적재
- 현재 raw 데이터를 다시 계산해 과거 판정을 만드는 백필은 하지 않음
- v0.6.2 이전처럼 권위 snapshot이 없는 버전은 만들거나 추정하지 않음
- core·aux·HY/BBB/A/CP 신용 노드·신용 에피소드·HY-BBB 렌즈의 당시 판정을 작은 행 단위 원장으로 저장
- `(cache_version, section, key)`를 고유키로 사용해 수동 재실행이나 재시도 시 중복 저장 방지
- 같은 고유키에 다른 내용이 들어오면 기존 기록을 조용히 덮어쓰지 않고 충돌로 처리
- 원장은 `versions/` 밖의 `audit/` 경로에 저장해 일반 캐시 prune과 분리
- 새 버전을 활성화한 뒤 **원장 보충을 먼저 끝내고 그다음 오래된 versions를 prune**
- 원장 보충이 실패하면 핵심 데이터 업데이트 성공은 유지하되 그 실행의 prune은 건너뛰어, 아직 옮기지 못한 권위 snapshot을 다음 refresh에서 다시 살릴 수 있게 함

보조 상태 파일 `audit/decision_ledger_status.json`에는 현재 원장 행 수, 보존된 권위 snapshot 버전 수, 마지막으로 추가된 버전, raw 재계산 여부를 기록합니다. v0.7.3은 아직 이 원장으로 성능 비율이나 위험 점수를 만들지 않습니다. 검증 보고서는 실제 기록이 충분히 쌓인 뒤 별도 버전에서 다룹니다.

## v0.7.2 최근 90일 기업 신용 변화

v0.7.2는 새 신용 엔진을 만들지 않습니다. v0.6.0부터 저장해 온 `credit_episode_nodes.parquet`와 `credit_episodes.parquet`를 읽기 쉬운 시간축으로 보여줍니다.

- 최신 신용 관측일을 포함한 **달력 최근 90일**을 표시
- 부담 상승 확인, 확정 뒤 상태 전환, 확인된 변화 안의 새 고점, 되돌림 시작, 정상화 확인만 사건으로 표시
- 확정되지 않고 사라진 `early_change`는 독립 사건으로 보여주지 않음
- 상승 변화가 실제로 확인된 경우에만 `확인되기 전부터 YYYY-MM-DD에 초기 변화가 보이기 시작했습니다`를 종속 참고 문장으로 표시
- 지난 기업 신용 변화 기록은 영구 누적 기록이라고 부르지 않고 **현재 제공되는 공식 자료로 다시 계산한 기록**이라고 표시
- 원자료 개정이나 이용 가능한 자료 범위 변화에 따라 과거 날짜와 변화가 기록된 시장이 조정될 수 있음을 지난 기업 신용 변화 기록 하단에 한 번만 안내
- 휴면 에피소드의 화면 종료일과 관찰 기간 계산을 같은 날짜 기준으로 통일
- 타임라인과 지난 기업 신용 변화 기록 모두 선행·후행, 인과 관계, 다음 위기 확률을 주장하지 않음

기존 상세 화면에서도 내부 `estimated_onset`을 `변화 시작`으로 직접 보여주지 않습니다. 현재 상태 상세에는 실제 상승 변화가 확인된 `confirmed_at` 날짜를 표시합니다.

v0.7.2와 함께 v0.7.0 표현 정리 패치의 기계 치환 흔적도 바로잡았습니다. `평소 변화가 나타난 곳`, `자료 변화가 나타난 곳`, `신용 변화 흐름가`, `현재 변화가 나타난 곳와 지속` 같은 문구를 자연스러운 `평소 범위`, `자료 범위`, `기업 신용 변화 흐름`, `현재 변화 범위와 지속 상태`로 정리했습니다.

## v0.7.1 장기금리 변화의 구성

v0.7.1은 v0.7.0의 변화 중심 UI를 유지하면서, 30년 금리 해석을 **30년짜리 금리끼리 나눠 보는 방식**으로 정리합니다. 새 위험점수나 새 핵심 상태는 만들지 않습니다.

- `DFII30`을 core·aux 판정과 분리된 전용 경로에서 refresh당 1회 수집
- `30년 일반·물가연동 국채금리 차이 = DGS30 - DFII30`을 공통 관측일에서 1회 계산
- 약 1개월(20관측)·약 3개월(60관측)의 `30년 미국 국채금리 = 물가 영향을 뺀 30년 금리 + 30년 일반·물가연동 국채금리 차이`를 첫 화면에 표시
- 30년 일반·물가연동 국채금리 차이는 **물가 기대·위험 등이 섞인 금리 차이**로만 표현하고 순수 기대인플레이션으로 단정하지 않음
- 10년 장기채 추가 보상은 30년 두 부분으로 나눈 값에 더하지 않고 **참고용**으로 표시
- 곡선 분류는 30년·2년 금리와 `30Y-2Y` 변화에 같은 20관측 창·10bp 유의폭을 사용
- 쉬운 한국어를 먼저 쓰고 `(bear steepening)` 같은 실제 시장 약칭은 괄호로 병기. 원인·인과 설명은 자동으로 붙이지 않음
- 기존 `DGS30 vs DFII10` 혼합 만기 구성 규칙 2개를 제거하고 10년 지표는 다른 만기의 참고 신호로 재정리
- 숨은 aux가 첫 화면 변화센터에 새는 기존 버그 수정. 상세 목록과 변화센터 노출 정책을 `AUX_CHANGE_CENTER_KEYS`로 분리하고 세 이벤트 채널을 slice 전에 필터. 현재 v0.7.1은 기존 사용자 노출을 유지하며 IGOAS만 제외

새 산출물은 `rate_composition_series.parquet`와 `rate_composition.json`입니다. 구버전 캐시에 이 파일이 없어도 앱은 기존 화면을 계속 읽고, 새 v0.7.1 refresh가 성공하면 동일 만기 구성 패널이 채워집니다.

## v0.7.0 변화 중심 대시보드

v0.7.0은 v0.6.2의 권위 있는 판정 스냅샷·diff 토대 위에서, 사용자가 **10초 안에 최근 변화와 다음 확인 대상을 파악**하도록 화면 구조를 바꿉니다. 판정 규칙·데이터 수집·Telegram은 바꾸지 않습니다.

- `한눈에 보기` — 오늘 한 줄 → 최근 갱신에서 달라진 것 → 아직 남아 있는 것 → 다음 확인 → 핵심 근거·가장 강한 반대 근거·확인 부족 순서
- 권위 있는 `decision_diff.json`을 사용자 언어로 렌더링해 시장 변화·데이터 상태 변화·복구 공백을 분리
- 핵심 6개를 모바일 카드로 표시하고 기본값은 `변화 있는 항목만 보기`; 평소 상태 항목은 필요할 때 전체 보기로 전환
- `기업 신용` 탭에 HY·BBB·A·CP 범위 지도를 추가. 화살표나 순서 가정 없이 현재 참여 범위와 상태만 표시
- 8개 상위 탭을 `한눈에 보기 / 기업 신용 / 흐름과 차트 / 비교 / 지표 설명` 5개로 통합
- 정상 운영 정보는 상단의 데이터 상태 한 줄로 축약하고, 상세 운영 진단은 전역 아코디언으로 이동
- `흐름과 차트`는 선택 지표 기준으로 30일 흐름과 전체 원자료 차트를 함께 보여주어 6개 차트를 한꺼번에 펼치지 않음
- 근거 보드는 개수를 세는 숨은 점수가 되지 않도록 핵심 근거 최대 2개, 가장 강한 반대 근거 최대 1개, 확인 부족 최대 2개만 압축 표시

`최근 갱신에서 달라진 것`은 **직전 권위 있는 정상 배치 대비**입니다. 로그인 없는 공유 Space에서 `내가 마지막으로 본 뒤`라고 과장하지 않습니다.


## v0.6.2 판정 기록 기반

v0.6.2는 첫 화면을 다시 바꾸기 전에 **당시 배치가 실제로 내린 판정을 권위 있는 기록으로 저장하고, 다음 배치와 안전하게 비교하는 토대**를 추가합니다.

- `decision_snapshot.json` — 핵심 6개, HY/BBB/A/CP, 신용 에피소드, HY-BBB 렌즈, 확인지표 방향, 실제 관측일과 데이터 상태를 당시 배치 기준으로 저장
- `decision_diff.json` — 직전 권위 있는 스냅샷과 비교해 시장 변화·데이터 상태 변화·복구 공백·판정 기준 경계를 분리
- `snapshot_format_version` — 나중에 필드가 추가돼도 필드 부재 자체를 시장 변화로 세지 않음
- 영역별 판정 schema — 핵심 상태·신용 에피소드·확인지표 규칙이 바뀐 영역만 비교 보류
- 관측일 전진 검사 — 같은 관측일에서 생긴 판정 차이를 시장 변화로 오보하지 않음
- 백필 금지 — v0.6.2 이전 캐시는 현재 코드로 재판정해 가짜 역사를 만들지 않음
- 복구 경계 — 수집 공백 뒤 복구값과 판정이 달라졌다면 정확한 이동 시점을 단정하지 않고 별도 복합 사건으로 기록
- 보존 정책 — 최근 90일 우선 + 최소 버전 수 + 안전 상한의 날짜·개수 이중 기준

`지난 30일 흐름`의 현재 규칙 기반 재구성은 그대로 유지하지만, **배치 diff는 그 재구성본을 절대 사용하지 않습니다.** 상단 `데이터 상태·운영 진단 보기`의 `판정 기록·변화 진단 보기`에서 스냅샷 포맷·영역별 schema·비교 상태·탐지 결과를 확인할 수 있습니다.

### 핵심 6개

- **주식시장이 예상하는 흔들림(VIX)** — 주식시장 예상 변동성
- **신용등급 낮은 기업의 추가금리(HY OAS)** — 위험도가 높은 기업의 회사채 부담 자체
- **10년 금리와 3개월 금리의 차이** — 경기보다 먼저 움직이는 금리 관계
- **미국 30년 국채금리** — 장기금리
- **미국 2년 국채금리** — 가까운 기준금리 예상에 민감한 금리
- **물가 영향을 뺀 10년 금리** — 물가 효과를 제외한 장기 금리 부담

### 기업 신용 렌즈: HY-BBB

`HY OAS - BBB OAS`는 별도 시장이 아닙니다. HY와 BBB를 다시 조합한 파생값이므로 기업 신용 범위 엔진의 독립 노드로 세지 않습니다.

이 렌즈는 다음 질문에만 답합니다.

> 현재 부담 증가가 투기등급 기업에 특히 집중되고 있는가, 아니면 투자등급 경계 기업도 함께 움직이는가?

### 확인 지표 5개

- **10년 구간 참고 맥락** — 10년 명목−실질 금리차, 10년 장기채 추가 보상
- **30년 금리 변화의 구성** — 첫 화면에서 30년 미국 국채금리·물가 영향을 뺀 30년 금리·30년 일반·물가연동 국채금리 차이를 같은 만기로 별도 표시
- **기업 부담이 어디까지 번졌나** — 투자등급 경계 기업의 추가금리(BBB), A등급 기업의 추가금리(A)
- **단기 기업자금시장도 영향을 받나** — 30일 A2/P2 비금융 CP 금리 - 30일 AA 비금융 CP 금리

### 외부 참고 2개

- **미국 금융시장 전반의 자금 사정(NFCI)**
- **미국 금융시장 전반의 불안(STLFSI)**

두 지표는 외부 공식 종합지표를 참고하는 용도입니다. 핵심 상태, 3개 영역 개수, 기업 신용 에피소드 참여, 조합의 지지·약화 판정에 넣지 않습니다.

## 기업 신용 범위·지속 엔진

v0.6.0의 가장 큰 변화입니다. 순서를 억지로 주장하는 `선행/후행 엔진` 대신 다음을 먼저 봅니다.

1. 현재 기업 신용 에피소드가 있는가
2. 실제 시장 중 어디가 참여하고 있는가
3. 각 시장의 변화가 새로 시작됐는가, 이어지는가, 되돌아오는가
4. 처음 벌어진 변화가 아직 얼마나 남아 있는가
5. 변화가 저신용 기업에 집중됐는가

### 실제 시장 노드

```text
회사채 시장
HY → BBB → A

다른 자금시장
CP
```

- **HY** — 위험도가 높은 기업의 회사채
- **BBB** — 투자등급 경계 기업의 회사채
- **A** — 투자등급 안쪽의 A등급 회사채
- **CP** — 단기 기업자금시장

과거 broad IG(`BAMLC0A0CM`)는 BBB를 포함하므로 BBB 뒤의 독립 확산 단계로 사용하지 않습니다. 구버전 캐시 호환과 운영 진단을 위해 수집 경로만 유지하고, 범위 엔진과 일반 사용자 상세 목록에서는 제외합니다.

### 노드 상태

- 평소 상태
- 초기 변화
- 상승 확인
- 상승 지속
- 되돌림
- 정상화

`초기 변화`는 관찰 단계이며 공식 참여 시장으로 세지 않습니다.

### 에피소드 상태

- **활성** — 최근 새 참여나 의미 있는 확대·상태 전환이 있음
- **휴면** — 과거 변화의 잔존은 남아 있지만 최근 의미 있는 활동은 없음
- **종료** — 참여 시장의 변화가 동결된 사전 기준선으로 충분히 되돌아옴

휴면 뒤 새 변화가 생기면 새 에피소드로 시작하고, 이전 에피소드의 잔존 변화가 있으면 별도 주석으로 남깁니다.

### 변화 시작과 잔존을 분리

```text
변화 강도
→ 후보 발생 · 추정 시작 · 확정

동결된 사전 기준선 대비 잔존 변화
→ 상승 지속 · 되돌림 · 정상화
```

오르는 속도가 멈췄다고 문제가 끝난 것으로 보지 않습니다.

확정 시 다음 값을 동결합니다.

- 추정 시작일
- 확정일
- 시작 직전 기준선

기준선은 추정 시작일 **직전** 유효 관측치들의 중앙값으로 잡아 초기 상승분이 섞이지 않게 합니다. 확정 뒤에는 소급 수정하지 않아 Telegram과 UI의 과거 설명이 뒤집히지 않게 합니다.

### 순서 표현 정책

v0.6.0은 사용자에게 실제 선행·후행을 주장하지 않습니다. 관련 타임스탬프는 처음부터 저장하지만 1차 산출물은 **범위·지속·잔존**입니다.

- `주식시장이 먼저 움직임` → **주식시장 쪽만 움직임**
- 지난 30일의 날짜 목록 → **새 변화가 처음 확인된 날짜**
- 날짜가 앞섰다는 사실만으로 원인·선행 관계를 만들지 않음

VIX는 신용 에피소드 노드가 아니라 `에피소드 시작 무렵 주식시장 불안도 있었는가`를 보는 맥락 정보입니다.

## ICE BofA 회사채 자료 정책

2026년 4월 이후 FRED에서 ICE BofA 회사채 계열은 최근 약 3년 자료만 제공합니다. v0.6.0은 이 현실을 숨기지 않습니다.

- HY는 **최근 3년 중 현재 위치**만 표시
- 실제 5년·10년 달력 범위가 없으면 5년·10년 위치를 계산하지 않음
- BBB·A 상세 화면은 `현재 확보된 자료 범위 안 위치`라고 표시
- 최근 3년을 `장기 역사`, `역사적으로 이례적`, `과거 위기 수준`이라고 부르지 않음
- 회사채 범위·지속 엔진은 최근 공식 자료 범위에서 현재 변화·지속·확산을 봄
- 장기 경제학적 의미와 과거 사례는 논문·공식 연구·사례 연구의 역할로 분리

HY의 기존 5년·10년 위치가 관측치 개수만으로 계산되던 문제도 수정했습니다. 이제 실제 달력 커버리지가 충분해야 해당 기간 백분위를 계산합니다.

## 지난 30일 요약과 흐름

핵심 6개의 과거 point-in-time 경로를 현재 규칙으로 재구성합니다.

- 한 달 사이 어떻게 달라졌나
- 한때 크게 움직였다가 되돌아온 변화
- 현재까지 남아 있는 추세
- 선택 지표의 지난 30일 값 변화
- 접어서 확인하는 현재 제공 전체 기간 흐름과 표

사용자 화면은 시간 경로를 정리할 뿐, 어느 지표가 원인이거나 먼저 움직였다고 주장하지 않습니다.

4. **흐름** — 지난 30일 요약, 선택 지표의 30일 차트, 접어서 보는 전체 기간·표·설명
5. **비교** — 최신 관측값 비교와 실제 공통 관측일 기준 같은 날짜 비교
6. **설명** — 지표 선택형 상세 설명, RiskRadar 읽는 법, 지표를 함께 보는 법

운영 정보는 독립 탭이 아니라 상단 `관리·진단` 아코디언에서 확인합니다.

## 데이터 파이프라인

```text
cron-job.org 수동 호출 또는 main의 version.py 변경
  → GitHub Actions refresh
  → run_refresh.py
  → FRED 핵심 6개 + 확인/외부/호환 지표 수집
  → 상태·3개 영역·조합·기업 신용 범위·지속 계산
  → HF Dataset versioned cache 저장
  → Telegram 알림

HF Space
  → HF Dataset의 최신 active cache를 읽기
  → FRED 직접 수집 없이 화면 표시
```

새 옵셔널 산출물:

```text
aux_raw.parquet
credit_episode_nodes.parquet
credit_episodes.parquet
```

옛 캐시에 새 파일이 없어도 UI 전체가 깨지지 않습니다. 모든 부속 파일은 하나의 `active_cache_version`에 고정해 읽습니다.

확인지표가 일시 실패하면 직전 실제 성공값을 이어 쓰되 최신성을 다시 계산합니다. 기업 신용 엔진에 필요한 BBB·A·CP 원자료도 직전 실제 원자료가 아직 오래되지 않았다면 복구해 경로를 유지합니다. 오래된 원자료는 현재 엔진 입력으로 사용하지 않습니다.

## 검증 도구

새 엔진의 임계값을 몇 개 위기 사례에 맞춰 튜닝하지 않습니다. 개발용 검증은 다음 세 축입니다.

- 평온기 오탐률
- 알려진 스트레스 국면의 상식적 해석
- 임계 백분위 ±2~3 변화에 대한 민감도

민감도는 에피소드 집합, 참여 시장, 시작일, 지속시간이 작은 임계값 변화에 과도하게 뒤집히지 않는지 확인하는 용도입니다.
